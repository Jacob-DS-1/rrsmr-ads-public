# Renewable weather-to-CF models (Task 2)

Production file: `era5_renewable_models.joblib` (``ClippedRegressor`` wrappers, predictions in [0, 1]).

## Selection

Three gradient boosting implementations were compared on a **temporal 80/20 train–test split**
(test metrics below). The model with the **highest test R²** was chosen for each energy type
(tie-break: lower RMSE, then faster training time). Each winner was **refit on the full**
`supply_model_training_ready.parquet` before export.

| Energy | Selected library | Role |
|--------|------------------|------|
| Wind | **XGBoost** | `wind_speed_100m_ms` → `wind_cf` |
| Solar | **LightGBM** | `ssrd_j_m2` → `solar_cf` |

## How Task 3-4 uses this model (Baseline + Weather Adjustment)

The model is **not** used to emit a raw CF curve directly. Instead Task 3-4
implements Priyanshi's Baseline + Weather Adjustment logic:

1. A **typical historic CF** library is computed from the training table by
   `month x hour x is_weekend` (median of observed `wind_cf` / `solar_cf`).
2. For each future hour and each of the **three representative weather years**
   (low / average / high wind), this model predicts a **weather-driven CF**
   from the scaffolded ERA5 input.
3. **Adjustment factor** = predicted CF / typical historic CF (clipped to a
   bounded range; safe-divide where typical CF is near zero).
4. **Unscaled MW** = baseline historic profile (`p50_mw`) x adjustment factor.

The model therefore captures *weather variance* on top of the historic shape;
the absolute annual energy is then re-anchored in Task 5 to the FES TWh target
for the matching technology.
